Tham số đầu tiên của route:
/{something} : truyền vào 1 path -> ở tham số thứ 2 là function: truyền tên biến đại diện cho path {something}
/{something?} : truyền hay không truyền cũng được, nhưng nếu gọi biến đại diện path trong hàm thì phải khai báo giá tị mặc định


Tạo controller bằng dòng lệnh
php artisan make:controller TestController

Tạo function func_hello trong class controller
	Tham số hàm là tham số của tham số thứ 2 trong hàm route
	Nội dung hàm là nội dung của hàm: tham số thứ 2 trong hàm route
Ở hàm route: đổi tham số thứ 2 là gọi tới hàm func_hello trong class TestController
	[\App\Http\Controllers\TestController::class, 'func_hello']
	Thay vì ghi đường dẫn của class dài như vậy, import ở đầu file

Từ controller gọi đến view:
Tạo file view 'Hello.php' nằm trong 'resources/views/' 
	Ở function func_hello trong class TestController trả về view:
		return view('Hello')
	Tuy nhiên, ở laravel khác so với mvc oop thuần, biến trong controller sẽ không gọi được qua file view
	Vì vậy, phải truyền biến vào khi gọi file view
		return view('Hello'. ['
			'tengido' => $ten,
			'tuoigido' => $tuoi
		'])
File view sẽ có nội dung "Hello $tengido, you are $tuoigido"

Ở file Route:
	URL ở trang chủ -> Gọi đến hàm formInput ở class HelloController
		FormInput: trả về view là 1 cái form
			FormInput.blade.php: (.blade giúp viết code php nhanh và gọn hơn)
			Để có thể truyền form dưới method post thì phải truyền kèm 1 đoạn token csrf
			@csrf = <input type="text" value="<?php echo csrf_token() ?>" name="_token">
			Action nhảy tới url /postData
Ở file route thêm 1 route khi url là /postData: nhảy tới hàm postData ở class HelloController
	Lưu ý: Vì dữ liệu truyền lên ở phương thức post -> Route::post
Hàm postData ở file HelloController:
	Tham số của hàm là 1 request: public function postData ( Request $request )
	Lấy ra nội dung của dữ liệu đã truyền lên: $content = $request->get('keycuarequest')
	Trả về 1 cái view hiển thị lên nội dung đó
	Show.blade.php: {{$content}}
